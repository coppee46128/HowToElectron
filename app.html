app
アプリケーションのライフサイクルを制御します。
アプリケーションの起動、終了などに関する挙動を設定できます。
その他にもそのOS特有の設定、chromiumの設定、web関連の設定も一部あります。

イベント

次の例は最後のウィンドウを閉じた時、アプリを終わらせる例を示しています。
イベントはメソッドを直接呼ぶのではなく、テキストとコールバックを渡します。

'ready'

Electronが初期化を終了したときに放出されます。

const {app,BrowserWindow}=require("electron");
let win;
app.on("ready",function(){
	win=new BrowserWindow();
	//win.loadURL('file://' + __dirname + '/index.html');
	//win.webContents.openDevTools();
});

'window-all-closed'

全てのウィンドウを閉じた時に呼ばれます。
もし全てのウィンドウを閉じてこのイベントを使っていない場合、デフォルトでアプリを終了させます。
ただしもし使っている場合アプリを終わらせるかどうか制御します。
もしユーザーがcmd+Qコマンド、また開発者がapp.quit()をした場合、
electronは最初に全てのウィンドウを閉じてwill-quitを発生させます。
この場合window-all-closedは呼ばれません。
const {app} = require('electron')
app.on('window-all-closed',function(){
	app.quit();
});

'before-quit'

戻り値：
  • event Event

アプリケーションがウィンドウを閉じる前に送出されます。

'will-quit'

戻り値：
  • event Event

すべてのウィンドウが閉じられ、アプリケーションが終了すると発生します。
will-quitイベントとwindow-all-closedイベントの違いについては、 window-all-closedイベントの説明を参照してください。

'quit'

戻り値：
  • event Event
  • exitCode Integer

ユーザーがアプリケーションでURLを開こうとしたときに発生します。 アプリケーションのInfo.plistファイルは、 CFBundleURLTypesキー内のURLスキームを定義し、 NSPrincipalClassをAtomApplication設定する必要があります。

このイベントを処理する場合は、 event.preventDefault()を呼び出す必要があります。

'activate' macOS

戻り値：
  • event Event
  • hasVisibleWindows Boolean

アプリケーションが起動されると発生します。 アプリケーションを初めて起動する、既に実行中のアプリケーションを再起動する、またはアプリケーションのドックやタスクバーのアイコンをクリックするなど、さまざまなアクションがこのイベントを引き起こす可能性があります。

'browser-window-blur'

戻り値：
  • event Event
  • window BrowserWindow

browserWindowが不鮮明になると発生します。

'browser-window-focus'

戻り値：
  • event Event
  • window BrowserWindow

browserWindowにフォーカスがあったときに送出されます。

'browser-window-created'

戻り値：
  • event Event
  • window BrowserWindow

新しいbrowserWindowが作成されると発生します。

'web-contents-created'

戻り値：
  • event Event
  • WebContents WebContents

新しいWebContentsが作成されると発生します。

メソッド

app.quit()

すべてのウィンドウを閉じるようにしてください。before-quitイベントが最初に出力されます。 すべてのウィンドウが正常に終了すると、 will-quitイベントが発行され、デフォルトではアプリケーションは終了します。

このメソッドは、すべてのbeforeunloadおよびunloadイベントハンドラが正しく実行されることを保証します。beforeunloadイベントハンドラでfalseを返すことによって、ウィンドウが終了をキャンセルする可能性があります。

app.exit([exitCode])

引数：
  • exitCode Integer（optional）

すぐにexitCodeで終了します。exitCodeデフォルトは0です。

すべてのウィンドウは、ユーザーに尋ねることなくすぐに閉じられ、 before-quitおよびwill-quitイベントは放出されません。

app.relaunch([options])

引数：
  • options Object（optional）
  • args String [] - （optional）
  • execPath String（optional）

現在のインスタンスが終了したときにアプリを再起動します。

デフォルトでは、新しいインスタンスは、現在のインスタンスで同じ作業ディレクトリとコマンドライン引数を使用します。 argsを指定すると、代わりにargsがコマンドライン引数として渡されます。 execPathが指定されると、 execPathは現在のアプリケーションの代わりに再起動のために実行されます。

このメソッドは実行時にアプリケーションを終了しないことに注意してくださいapp.exitを呼び出してappを再起動すると、 app.quitまたはapp.exitを呼び出すapp.relaunchがあります。

app.relaunchが複数回呼び出されると、現在のインスタンスが終了した後で複数のインスタンスが開始されます。

直ちに現在のインスタンスを再起動し、新しいインスタンスに新しいコマンドライン引数を追加する例

const {app} = require('electron')

app.relaunch({args: process.argv.slice(1).concat(['--relaunch'])})
app.exit(0)

app.isReady()

戻り値：
  • Boolean - Electronが初期化を終了した場合はtrue返し、そうでない場合はfalse返します。

app.focus()

Linuxでは、最初に表示されるウィンドウに焦点を当てます。macOSでは、アプリケーションをアクティブなアプリケーションにします。Windowsでは、アプリケーションの最初のウィンドウに焦点を当てます。

app.hide()macOS

アプリケーションウィンドウを最小化せずに非表示にします。

app.show()macOS

アプリケーションウィンドウが非表示になった後に表示されます。 自動的にフォーカスを合わせません。

app.getAppPath()

戻り値：
  • String - 現在のアプリケーションディレクトリです。

app.getPath(name)

引数：
  • name String - 関連付けられた特別なディレクトリまたはファイルへのパスです。 失敗すると、 Errorがスローされます。

戻り値：

nameで次のパスを要求することができます：

  • home ユーザーのホームディレクトリ。
  • appData ユーザーごとのアプリケーションデータディレクトリ。デフォルトでは次のものを指します。
    • Windowsでの%APPDATA%
    • Linuxでは$XDG_CONFIG_HOMEまたは~/.config
    • ~/Library/Application Support
  • userData アプリケーションの設定ファイルを格納するディレクトリです。デフォルトでは、 appDataディレクトリにはアプリ名が追加されています。
  • temp 一時ディレクトリ。
  • exe 現在の実行可能ファイル。
  • module libchromiumcontentライブラリ。
  • desktop 現在のユーザーのデスクトップディレクトリ。
  • documents ユーザーの「マイドキュメント」のディレクトリ。
  • downloads ユーザーのダウンロード用のディレクトリ。
  • music ユーザーのmusicディレクトリ。
  • pictures ユーザーのpicturesディレクトリ。
  • videosユーザーのvideosディレクトリ。
  • pepperFlashSystemPlugin Pepper Flashプラグインのシステムバージョンへのフルパス。

app.getFileIcon(path[, options], callback)

引数：
  • path String
  • options Object（optional）
    • size String
      • small - 16×16
      • normal - 32x32
      • large - Linuxでは48x48、 Windowsでは32x32、 MacOSではサポートされません。
  • callback関数
    • error Error
    • icon NativeImage

パスの関連アイコンを取得します。

Windowsでは、2種類のアイコンがあります。

  • .mp3 、 .pngなどの特定のファイル拡張子に関連付けられたアイコン
  • .exe 、 .dll 、 .icoなどのファイル内のアイコン。

    • LinuxおよびmacOSでは 、アイコンはファイルmimeタイプに関連付けられたアプリケーションに依存します。

    app.setPath(name, path)

    引数：
    • name String
    • path String

    name関連付けられた特別なディレクトリまたはファイルへのpathをオーバーライドしpath 。 パスに存在しないディレクトリが指定されている場合、この方法でディレクトリが作成されます。 失敗すると、 Errorがスローされます。

    app.getPath定義されたnameパスのみを上書きできます。

    デフォルトでは、WebページのCookieとキャッシュはuserDataディレクトリに格納されます。 この場所を変更ready場合は、 appモジュールのreadyイベントが発生する前にuserDataパスをオーバーライドする必要があります。

    app.getVersion()

    戻り値：
    • String - ロードされたアプリケーションのバージョンです。
    アプリケーションのpackage.jsonファイルにバージョンが見つからない場合は、 現在のバンドルまたは実行可能ファイルのバージョンが返されます。

    app.getName()

    戻り値：
    • String - 現在のアプリケーションの名前。アプリケーションのpackage.jsonファイル内の名前です。

    通常、 package.jsonのnameフィールドは、npmモジュールの仕様に従って、短い小文字の名前です。 また、通常、アプリケーションの完全な大文字の名前であるproductNameフィールドも指定する必要があります。 これはElectronよりも優先されます。

    app.setName(name)

    引数：
    • name String

    現在のアプリケーションの名前を上書きします。

    app.getLocale()

    戻り値：
    • String - 現在のアプリケーションロケールです。 可能な戻り値については、ここで説明します 。
    注：パッケージ化されたアプリケーションを配布するときには、 localesフォルダも同梱する必要があります。

    注： Windowsでは、 readyイベントが発生した後に呼び出す必要があります。

    app.addRecentDocument(path)macOSWindows

    引数：
    • path String

    最近のドキュメントリストにpathを追加しpath 。

    このリストはOSによって管理されます。Windowsでは、タスクバーからリストにアクセスでき、macOSではドックメニューからアクセスできます。

    app.clearRecentDocuments()macOSWindows

    最近のドキュメントリストを消去します。
